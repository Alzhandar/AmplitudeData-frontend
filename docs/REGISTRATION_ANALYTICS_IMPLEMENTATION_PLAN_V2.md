## Фаза 1 — Фронтенд (без изменений бэкенда)

### Что реализуем

Используем **два параллельных вызова** существующего endpoint:
1. Запрос за **текущий период** (выбранный пользователем)
2. Запрос за **предыдущий аналогичный период** (автоматически сдвинутый)

Из них вычисляем на клиенте:
- Дельту: `текущий − предыдущий`
- Процент изменения: `(текущий − предыдущий) / предыдущий × 100 %`
- Средний в день: `registrations / кол-во дней периода`

**Ограничение:** оба периода должны попадать в один календарный год (ограничение
существующего бэкенда). При выборе 90 дней с января предыдущий период автоматически
обрезается по 01.01.

### Изменения в коде

#### 1. `hooks.ts` — расширить хук

Новый хук `useMobileRegistrationsComparison(startDate, endDate, enabled)`:
- Вычисляет `prevStartDate` / `prevEndDate` = сдвиг на длину текущего периода назад
- Делает два параллельных вызова через `Promise.all`
- Возвращает `{ current, previous, delta, deltaPercent, avgPerDay, loading, error }`

```ts
// псевдокод
const days = diffDays(startDate, endDate);
const prevEnd = dayBefore(startDate);
const prevStart = subtractDays(prevEnd, days - 1);
// Если prevStart < January 1 of current year → clamp to Jan 1
```

#### 2. `RegistrationsBlock.tsx` — полная замена

Компонент становится полностью самодостаточным:
- Хранит `preset` (7д/30д/90д) во внутреннем state
- Вычисляет даты из пресета
- Вызывает `useMobileRegistrationsComparison`
- Рендерит дельту с цветом: зелёный ↑ / красный ↓ / серый =

#### 3. Пресеты периода

```ts
type Preset = "7d" | "30d" | "90d";
```

Пресет `ytd` (с начала года) убирается из-за ограничения — для него нет предыдущего
года, сравнение будет некорректным. Можно вернуть позже в Фазе 2.

## Фаза 2 — Спарклайн (требует изменений бэкенда)

### Что нужно от бэкенда

Новый endpoint: `GET /api/amplitude/mobile-registrations-daily/?year=&start_date=&end_date=`

Ответ:
```json
{
  "days": [
    { "date": "2026-04-01", "count": 38 },
    { "date": "2026-04-02", "count": 55 },
    ...
  ]
}
```

### Что добавляем на фронтенде

- Мини-спарклайн (SVG, без сторонних библиотек) — 1–2 строки значений
- Пиковый день с подписью

### Когда

После согласования Фазы 1 и при наличии ресурса на бэкенд.
