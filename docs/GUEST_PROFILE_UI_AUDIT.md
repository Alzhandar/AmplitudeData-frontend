## 2. Детальный разбор проблем

### 2.1 Структура страницы — отсутствие вкладок (КРИТИЧНО)

**Что сейчас:**  
Единый вертикальный скролл: Поиск → Карточка гостя → История покупок → Cashback → Crystals → Мобильная активность.  
При 20 покупках + 50 кешбэк-операций страница вырастает в бесконечный список.

**Как в проекте:**  
Другие страницы (bonus-transactions) логически разделяют «форму» и «историю» двумя `<section>`, но это работает, потому что там одна таблица.

**Что надо сделать:**  
После нахождения гостя показывать **вкладки (tabs)**:
```
[Обзор] [История покупок] [Cashback] [Crystals] [Мобильная активность]
```
- Вкладка **Обзор** — карточка гостя + 4 balance-карточки + предупреждения.  
- Остальные вкладки — соответствующие таблицы с пагинацией.  

Реализовать через локальный state `activeTab`, без роутинга — это инструмент на одной странице.

---

### 2.2 Детали покупки — сырой JSON (КРИТИЧНО)

**Что сейчас:**  
```tsx
<pre className="mt-2 max-h-52 overflow-auto rounded bg-slate-900 p-2 text-xs text-slate-100">
  {JSON.stringify(items, null, 2)}
</pre>
```
Пользователь видит:
```json
[
  {
    "id": 12345,
    "time_create": "2026-03-01T...",
    "order": 99999,
    "name": "Пицца Маргарита",
    ...
  }
]
```

**Что надо сделать:**  
Открывать **Modal** (компонент уже есть в проекте — `Modal.tsx`) с двумя чистыми таблицами:

**Товары:**  
| Название | Кол-во | Цена | Скидка | Итог |

**Оплаты:**  
| Тип оплаты | Сумма | Cashback-оплата |

---

### 2.3 Загрузка — отсутствие Skeleton

**Что сейчас:** Пустой экран → резкое появление всего контента.

**Как в проекте:**  
`Skeleton` и `SkeletonText` из `@/features/common/components/ui/Skeleton` используются в bonus-transactions для плейсхолдеров истории задач.

**Что надо сделать:**  
Пока `loading === true` — показывать:
- Карточка гостя: 4 skeleton-плашки вместо balance-карточек.
- Таблица: `SkeletonText` rows (5–8 строк).

---

### 2.4 Балансы — форматирование чисел

**Что сейчас:** `38285` (голое число без форматирования и единиц).

**Что надо сделать:**  
```ts
function formatCurrency(n: number): string {
  return new Intl.NumberFormat("ru-RU").format(n) + " ₸";
}
function formatCrystals(n: number): string {
  return new Intl.NumberFormat("ru-RU").format(n) + " 💎";
}
```

Карточки баланса переделать в наглядный grid с иконками:

```
┌──────────────────┐  ┌──────────────────┐
│  💰 Cashback      │  │  💎 Crystals      │
│  38 285 ₸        │  │  1 200            │
│  Сгорает: 11 мая │  │                  │
└──────────────────┘  └──────────────────┘
```

---

### 2.5 Статус гостя — не используется StatusBadge

**Что сейчас:**  
Хардкодный inline-badge:
```tsx
className={`inline-flex items-center rounded-full px-3 py-1 text-xs font-semibold ${
  guestStatus?.is_blocked ? "bg-rose-100 text-rose-700" : "bg-emerald-100 text-emerald-700"
}`}
```

**Как в проекте:**  
`StatusBadge` поддерживает `completed` (зелёный) и `failed` (красный), но привязан к статусам задач. 

**Что надо сделать:**  
Расширить `StatusBadge` вариантом `active`/`blocked`, или создать `GuestStatusBadge` по аналогии.

---

### 2.6 Предупреждения о частичных данных

**Что сейчас:**  
Amber-блок живёт в карточке гостя. Если cashback history недоступен, это видно только там — пользователь может не заметить, что таблица Cashback пуста из-за ошибки, а не потому что история пустая.

**Что надо сделать:**  
- Предупреждения показывать **в соответствующей вкладке** рядом с пустой таблицей.
- Добавить кнопку ✕ для демиссии каждого предупреждения.
- Использовать стиль из проекта: amber bordered card + иконка предупреждения.

---

### 2.7 Пагинация

**Что сейчас:**  
Показывается первые 50 из 725 cashback-записей. Пользователь не знает, что это не всё.

**Что надо сделать:**  
Под таблицей добавить строку:
```
Показано 50 из 725 записей
```
В первой версии — это достаточно. В расширенной — добавить кнопку «Загрузить ещё».

---

### 2.8 Мобильная активность

**Что сейчас:**  
Колонка «Устройство» = `device_id` (например `ANDROID-abc123xyz...`).  

**Что надо сделать:**  
- Колонка «Устройство» = `device_model` или `device_brand + device_model` из `MobileSession`.  
- `device_id` убрать из таблицы или перенести в tooltip.
- Backend `MobileActivityService` уже возвращает `platform`, нужно добавить `device_model`.
